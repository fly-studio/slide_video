# 任务完成工作流程

## 代码开发完成后的标准流程

### 1. 代码格式化
```bash
# 使用ruff或black格式化代码（待配置）
ruff format .
```

### 2. 代码检查
```bash
# 静态类型检查（待配置）
mypy .

# Linting检查（待配置）
ruff check .
```

### 3. 运行测试
```bash
# 单元测试（待实现）
pytest

# 或使用unittest
python -m unittest discover
```

### 4. 功能测试
```bash
# 运行主程序验证功能
python main.py
```

### 5. 代码审查要点
- [ ] 是否遵循项目代码风格
- [ ] 是否添加类型注解
- [ ] 是否使用中文注释
- [ ] 是否应用了缓动效果（CSS3兼容）
- [ ] 分辨率转换是否正确
- [ ] 是否避免代码重复

## Git工作流（如需要）
```bash
# 查看状态
git status

# 添加更改
git add .

# 提交（需用户明确要求）
git commit -m "描述性提交信息"

# 推送（需用户明确要求）
git push
```

⚠️ **重要提醒**: 
- 不要在用户未明确要求的情况下执行git操作
- 危险操作（删除、强制推送等）需要二次确认
